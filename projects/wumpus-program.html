Wumpus Program

A classic C-based survival game developed as my introductory project for CS 211 at UIC. Navigate a treacherous 20-room cave, avoid hidden hazards, and hunt the elusive Wumpus using limited resources and logic.

Overview

The Wumpus Program served as my gateway into low-level programming during my first semester at UIC. The project was developed in two distinct phases: the initial build established the core movement and hazard logic (pits and the Wumpus), while the second phase introduced complex mechanics like "super-bats" and a guided arrow system.

While the course was originally hosted on Repl.it, the project challenged me to think deeply about how data lives in memory. Beyond the logic of the game, I spent significant time balancing the "fun factor"—ensuring the cave layout was navigable while keeping the threat of the Wumpus unpredictable and engaging.

The game is controlled via a CLI (Command Line Interface) with the following primary inputs:

  • Move: M [ROOM_NUM] (e.g., M 3)
  • Shoot: S [NUM_ROOMS] [ROOM_1] [ROOM_2] [ROOM_3]
  • Map/Directions: P or D
  • Cheat Mode: C (Displays all hazard locations)
  • Guess Wumpus: G (Final attempt to win)

Key Features

  • Procedural World Generation: Randomizes positions for the player, Wumpus, pits, bats, and the arrow at the start of every game.
  • Dynamic Hazard System: Includes "Super-bats" that transport players to random rooms and bottomless pits that result in an immediate game over.
  • Sensory Feedback: Provides environmental clues (stench, drafts, rustling) based on the proximity of hazards to the player's current room.
  • Guided Arrow Mechanics: A complex shooting system allowing arrows to travel through up to three rooms with custom ricochet logic.

Technical Details

  • Memory Management: Utilized malloc and sizeof for dynamic array allocation to handle room sequencing and unique position generation.
  • Graph Representation: Implemented the 20-room cave system using a 2D adjacency array to define valid movement paths between nodes.
  • Struct-Driven State: Managed game data (positions, move counts, and item status) using a custom GameInfo typedef for cleaner data passing.
  • Input Validation: Developed a robust command parser using switch statements and toupper() to handle case-insensitive user inputs.

Challenges & Solutions

The Problem: The primary hurdle was designing a non-linear map navigation system using 20 interconnected "rooms." Initially, I struggled with manual memory allocation using malloc, which led to frequent segmentation faults and unstable builds when trying to store neighbor room data.

The Solution: I pivoted from a purely dynamic heap-allocated approach to a structured 2D adjacency array. This allowed for O(1) lookups of adjacent rooms while ensuring the program’s memory footprint remained stable. By wrapping the player and hazard states into a single GameInfo struct, I was able to pass the entire game "world" through functions by reference, significantly reducing code complexity and debugging time.

What I Learned

  • Memory Management: Navigated the differences between Heap and Stack allocation. I originally used malloc to handle room arrays, learning how to manually manage memory and avoid segmentation faults.
  • Static vs. Dynamic Allocation: Gained the intuition to recognize when a static array is more efficient than dynamic allocation—specifically for fixed-size data structures like the 20-room cave map.
  • Pointer Arithmetic: Mastered the use of pointers to traverse 2D arrays and update game state by reference within the GameInfo struct.
  • Edge Case Handling: Developed logic to handle "ricochets" and unique random spawns, ensuring the Wumpus, hazards, and player never overlap at the start of a game.

Future Improvements

  • Implement a BFS (Breadth-First Search) algorithm to ensure the player never spawns in an inescapable position.
  • Transition from a 2D adjacency array to a linked-list-based Graph data structure for more complex cave shapes.
  • Add a persistent high-score system using file I/O to track the fewest moves taken to slay the Wumpus.
