Wordle Game Guesser

An automated C-based solver that utilizes recursive elimination and constraint logic to "crack" any 5-letter secret word in a simulated Wordle environment.

Overview

The Wordle Game Guesser is an intelligent agent designed to play Wordle autonomously. Unlike a human player who relies on vocabulary intuition, this program uses a process of elimination. By analyzing the "feedback" from each guess (Greens, Yellows, and Grays), it systematically prunes a dictionary of over 12,000 words until only the secret word remains.

The project simulates a full game loop, allowing users to either provide a specific secret word or let the computer pick one at random, then watching the "thinking" process as it narrows down the possibilities in real-time.

Key Features

  • Automated Feedback Loop: Implements a visual "asterisk and capitalization" system to represent near-misses and exact matches, mimicking the original game's UI.
  • Dynamic Word Pruning: A powerful cancelWords engine that filters the dictionary based on four distinct scoring states (Exact Match, Positional Misplace, Non-existence, and Duplicate Handling).
  • Randomized Testing: Integrated srand and time hooks to allow for randomized secret word selection across multiple game rounds.
  • Alphabetical Tie-Breaking: Uses a dual-stage comparison function to ensure consistent behavior when multiple words share the same validity score.

Technical Details

  • Constraint Logic: Developed a multi-pass filtering system. It handles the "Double Letter" edge case—ensuring that if a guess has two 'E's but the secret word only has one, only one 'E' is marked as a match.
  • State Management via Scores: Instead of deleting data, the program uses a "Score of -1" as a flag to ignore invalid words, which is then handled efficiently by qsort.
  • Memory Safety: Utilized malloc for temporary string copies during comparison to ensure the original dictionary remains intact for subsequent rounds of the game.

Challenges & Solutions

The Problem: Handling duplicate letters was a major challenge. If the guess was "SPEED" and the secret word was "ABIDE," the program would initially try to mark both 'E's as misplaced (Yellow), which is logically incorrect.

The Solution: I implemented a "Character Cancellation" strategy. By creating temporary copies of the words and replacing matched characters with underscores (_) in a specific order (Greens first, then Yellows), I ensured each letter in the secret word is only "claimed" once per guess.

What I Learned

  • Algorithmic Pruning: Learned how to translate game rules into strict logical constraints that can be applied to large datasets.
  • Global Definitions: Utilized #define macros for WORD_LENGTH and MAX_NUMBER_OF_WORDS to create a more maintainable and readable codebase.
  • Data Transformation: Mastered the use of tolower() and toupper() to provide visual feedback while maintaining the underlying data's integrity.
