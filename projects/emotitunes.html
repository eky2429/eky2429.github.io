Emotitunes

A website that prompts the user with a series of questions to determine their mood and then suggests songs that match that mood.

  • HTML/CSS/JS
  • Node.JS
  • Visual Studio Code
Live Demo

Overview

This was the ONLY PROJECT I has for User Interface Design and Programming (CS 422)! It was a very long project that involved a lot of planning and designing (mainly iterative design). My group members and I conducted user studies to determine what features users wanted in a mood-based music recommendation system. After gathering user feedback, we created wireframes and prototypes of the website using Figma. Once we had a solid design, we implemented the website using HTML, CSS, and JavaScript. The final product was a functional website that allowed users to input their mood and receive song recommendations based on their responses.

Key Features

  • A simple 10 question quiz that would determine the user's mood and generate a playlist
  • A way to save and share playlists on Spotify
  • Settings to store user preferences (theme, font size, and font color)

Technical Details

  • Implemented essential classes for an interpreter: Token, Lexer, Parser, and main script
  • Separated Parser logic from main script and command(s) logic
  • Implemented a Command Design System so each command class has a constrctor and execute()

Challenges & Solutions

One challenge I faced was figuring out how to use pointers to store data, when I asked ChatGPT what to do about it (which was allowed in the class and something O suually avoid), it suggested using vectors to store data instead of raw pointers, which simplified memory management and reduced the risk of memory leaks. This approach allowed me to focus more on the interpreter logic rather than low-level memory handling.

What I Learned

  • Utilizing AI for debugging and problem-solving
  • Benefits of using vectors instead of raw pointers
