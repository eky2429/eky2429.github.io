Best Wordle Word

A high-performance C utility that analyzes dictionary datasets to determine the mathematically optimal starting and follow-up words for Wordle.

Overview

Building on my experience with procedural logic, I developed this program to solve a data-driven problem: "What is the best possible first move in Wordle?" The program parses two distinct datasets—official answers and allowed guesses—to calculate scores based on letter frequency and positional accuracy.

The core of the project is a comparison engine that simulates thousands of "guesses" against the answer key, ranking words based on how much information they reveal to the player.

Key Features

  • Dual-File Parsing: Separates "Answer" lists from "Guess" lists to mirror the actual Wordle game constraints.
  • Heuristic Scoring Engine: Assigns weighted values (3 points for "Green" matches, 1 point for "Yellow" matches) to rank word effectiveness.
  • Multi-Step Strategy: Beyond the first word, the program can calculate the best *second* word by excluding letters already "used" by the top-tier starting word.
  • Dynamic Menu System: Allows users to swap datasets on the fly and toggle between single-word or two-word strategy analysis.

Technical Details

  • Heuristic Scoring: Developed a custom scoring algorithm that simulates game logic by weighting Green (3pts) and Yellow (1pt) matches to find statistically superior starting words.
  • Nested Loop Processing: Implemented a strategy-ranking system that re-evaluates the entire word library (O(N²) complexity) to find the best second-word follow-up.
  • Memory Safety: Rigorously managed heap memory using malloc and free within nested structs, ensuring zero memory leaks during long-running data processing.
  • Advanced Sorting: Implemented qsort with a custom comparator function to handle primary sorting (by score) and secondary sorting (alphabetical) simultaneously.
  • Memory Management: Utilized a wordCountStruct with dynamically allocated strings to store and rank thousands of potential words without memory leaks.
  • Positional Logic: Developed a two-pass algorithm to correctly identify "Green" matches before calculating "Yellow" matches, preventing the over-counting of duplicate letters.

Challenges & Solutions

The Problem: The "Best Second Word" calculation required comparing every potential guess against every possible answer for every top-tier first word. This created a significant computational load when processing larger dictionary files.

The Solution: Instead of focusing on raw execution speed, I prioritized logic accuracy and memory stability. I developed a "cancellation" system using character masking to ensure the second-word suggestions were mathematically distinct from the first. While the process took several minutes for large datasets, the result was a highly accurate strategy that consistently identified the most efficient path to a win.

The Problem: Scoring the "Best Second Word" required a massive amount of computation, as the program had to re-evaluate the entire dictionary against every top-scoring first word.

The Solution: I implemented a "letter cancellation" system using underscores (_) to temporarily mask letters in the answer key. By passing the top word into the scoring function first, I was able to "filter" the answers, allowing the program to accurately find follow-up words that provide the most *new* information.

What I Learned

  • Struct Pointers: Deepened my understanding of typedef and pointer-to-struct arrays for managing complex datasets.
  • Algorithm Efficiency: Learned the importance of minimizing file I/O by reading data into memory once and processing it locally.
  • Complexity Management: Managed the logic of nested loops and conditional scores to ensure the "Best Second Word" didn't produce redundant suggestions.
